# Overview

This project builds on a previous console bookstore app by replacing the JSON-based data storage with a SQLite relational database. It demonstrates how to connect to a database, build SQL commands in C#, and perform full CRUD operations (insert, query, and delete) against a persistent local database.

Key concepts practiced:
- Setting up and initializing a SQLite database from C#
- Creating tables with `CREATE TABLE IF NOT EXISTS`
- Seeding initial data with `INSERT` statements
- Querying records with `SELECT` and reading results via `SqliteDataReader`
- Inserting user-provided records using parameterized queries to prevent SQL injection
- Deleting records by ID with `DELETE WHERE`
- Using `Microsoft.Data.Sqlite` NuGet package


# Relational Database

This project uses **SQLite**, a lightweight, file-based relational database. The database is stored locally as `books.db` and is created automatically at runtime if it does not already exist. SQLite was chosen because it requires no server setup and integrates easily with C# via the `Microsoft.Data.Sqlite` package.

**Table: Books**

| Column | Type    | Constraints                |
|--------|---------|----------------------------|
| Id     | INTEGER | PRIMARY KEY, AUTOINCREMENT |
| Title  | TEXT    | NOT NULL                   |
| Author | TEXT    | NOT NULL                   |
| Genre  | TEXT    | NOT NULL                   |
| Price  | REAL    | NOT NULL                   |
| Stock  | INTEGER | NOT NULL                   |

The `Books` table stores the bookstore's inventory. Each row represents a unique title with its author, genre, price, and current stock count. The `Id` column is auto-generated by SQLite and is used to identify records when updating or deleting.
